# Coding Practices

This track establishes good habits for writing research code that is readable, testable, and reproducible.
You will learn how to structure a Python project, write clear documentation, and build a test suite.

!!! note "Under construction"
    Lesson content is being developed. The outline below shows what each lesson will cover.

## Lessons

- **Lesson 01 — Project Layout**: How to organise files and modules in a Python project.
- **Lesson 02 — Documentation**: Writing NumPy-style docstrings and building a documentation site with MkDocs.
- **Lesson 03 — Testing**: Writing unit tests with pytest and understanding what to test.
- **Lesson 04 — Packaging**: Using `pyproject.toml` and `pip` to make a project installable.
- **Lesson 05 — Linting and Formatting**: Automating code quality with ruff and pre-commit.
